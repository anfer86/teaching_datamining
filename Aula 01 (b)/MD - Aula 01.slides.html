O que isto tem a ver com Minerao de Dados?

  • A produo de dados pelos usurios de Internet gigante

  • O Google utiliza (e cria) tcnicas de minerao de dados, como Recuperao de Informao (Information Retrieval) para recuperar resultados das nossas consultas. Isto mais do que uma consulta SQL.

  • Com os dados do Twitter possvel ver quais so os hot topics nas diferentes regies do mundo e inferir diferentes sentimentos dos usurios (ou grupos de usurios) usando tcnicas de minerao de textos (text mining), chamada de anlise sentimento (sentiment analysis)

  • O Facebook usa minerao de dados para aprender sobre fake profiles e poder dar um grau de probabilidade de que um perfil de facebook trata-se de um fake. A classificao um exemplo de tcnica utilizada para esse fim.

Processo de Descoberta de Conhecimento em Bases de Dados¶

  • Tambm chamado de KDD (Knowledge Discovery in Databases), foi proposto por Piatetski-Shapiro em 1989

  • Esse processo tem como principal objetivo extrair padres intrnsecos nos dados e apresent-los de forma a facilitar a sua assimilao como conhecimento, bem como da utilizao para outros fins

Etapas do Processo de Descoberta de Conhecimento (KDD)¶

Pré-processamento¶

  • Visa preparar os dados para uma anlise eficiente e eficaz

  • Algumas tarefas de pr-processamento:

    • Limpeza (remoo de rudos e dados inconsistentes)

    • Integrao (combinao de dados obtidos a partir de mltiplas fontes)

    • Seleo de Dados ou Reduo (escolha dos dados relevantes para anlise)

    • Transformao (transformao dos dados para um formato adequado para anlise)

Mineração de Dados ou Extração de Padrões¶

  • Aplicao de algoritmos capazes de extrair padres e/ou informaes e conhecimento relevantes a partir dos dados pr-processados

  • Algumas tarefas de minerao:

    • Anlise descritiva estatstica

    • Agrupamento

    • Modelos de Predio (classificao e regresso)

    • Associao (extrao de regras de associao)

    • Deteco de Anomalias

Pós-procesamento ou avaliação de modelos¶

  • Avaliao da qualidade dos modelos ou conhecimento extrado na etapa de Minerao de Dados

  • Algumas tarefas de ps-processamento:

    • Verificar a preciso dos modelos construdos em novos dados

    • Identificar conhecimentos verdadeiros e conhecimentos triviais

    • Identificar problemas no pr-processamento ou minerao que precisem ser modificados para a melhora do processo de descoberta de conhecimento

Aprendizado Supervisionado e Não-Supervisionado¶

As tcnicas para aprender modelos a partir de dados so classificadas de acordo com o tipo de aprendizado em:

  • Aprendizado Supervisionado

  • Aprendizado No-supervisinado

  • Aprendizado Semi-Supervisionado

Aprendizado Supervisionado¶

No aprendizado supervisionado os exemplos esto constitudos por atributos, onde um desses atributos classe

O atributo classe permite orientar o processo de aprendizando para ajustar um modelo aos dados

Exemplo Atributo 1 Atributo 2 ... Atributo m Atributo classe
$e_1$ $x_{1,1}$ $x_{1,2}$ ... $x_{1,m}$ $y_1$
$e_2$ $x_{2,1}$ $x_{2,2}$ ... $x_{2,m}$ $y_2$
... ... ... ... ... ...
$e_n$ $x_{n,1}$ $x_{n,2}$ ... $x_{n,m}$ $y_n$

Neste aprendizado o objetivo consiste em encontrar uma funo $f(X) = \hat{y}$, em que, dado os valores dos $m$ atributos, estime $y$.

As tarefas de interesse do aprendizado supervisionado so preditivas e definidas por:

  • Classificao: quando o valor da classe um valor categrico (ou discreto)
  • Regresso: quando o valor da classe um valor numrico.

Aprendizado Não-superivsionado¶

No aprendizado no-supervisionado os exemplos esto constitudos por atributos, mas no h um atrbiuto classe

Isso indica que no tem como orientar o processo de aprendizado tentando acertar algum valor de atributo especfico

Exemplo Atributo 1 Atributo 2 ... Atributo m
$e_1$ $x_{1,1}$ $x_{1,2}$ ... $x_{1,m}$
$e_2$ $x_{2,1}$ $x_{2,2}$ ... $x_{2,m}$
... ... ... ... ...
$e_n$ $x_{n,1}$ $x_{n,2}$ ... $x_{n,m}$

Neste tipo de aprendizado tentamos encontrar grupos de dados que apresentem comportamento similar

As tarefas de interesse dentro do aprendizado no-supervisionado so:

  • Agrupamentos: agrupar exemplos de acordo com algum critrio de distncia ou similaridade
  • Extrao de regras de associao: encontrar relaoes entre atributos e os valores dos atributos

Aprendizado Semi-Supervisionado¶

  • No aprendizado semi-supervisionado apenas alguns exemplos do conjunto de dados possuem o valor do atributo classe.

  • Muitas vezes muito caro classificar elementos. Alguns exemplos:

    • Transcrever segmentos de udio
    • Identificar observaes de experimentos fsicos
    • Identificar os rostros de fotos do Facebook manualmente

  • Nestas situaes, com algumas instncias rotuladas (exemplos classificados) possvel inferir os rtulos e classes de outras instncias.

Fonte: Wang, F., & Zhang, C. (2007). Robust self-tuning semi-supervised learning. Neurocomputing, 70(16-18), 2931-2939.

Paradigmas de Aprendizado¶

Alguns autores chamam de paradigmas os aprendizados supervisionado e no-supervisionado (CASTRO e FERRARI, 2016).

No entanto a literatura clssica divide os paradimgas de aprendizado da seguinte forma:

  • Simblico
  • Baseado em Instncia
  • Estatstico
  • Conexionista
  • Gentico
